APPENDIX: RULE REFERENCE
================================================================================

ADR-006: Code Style & PSScriptAnalyzer Rules
  - No Write-Host usage (use Write-Output, Write-Verbose, Write-Error, Write-Log)
  - No -ForegroundColor usage
  - 4-space indentation only (no tabs)
  - K&R bracing style (opening brace on same line)

CLAUDE.md Regel 3.1a: ASCII-only Output Strings
  - All output strings use ONLY ASCII characters
  - NO: °, ✓, ✗, •, █, ░, →, ←, ⏳, ä, ö, ü, etc.
  - YES: C (instead of °C), [OK]/[FAIL], *, -, #, >, <

STRUCTURE.md Regel 7.5: Line Length
  - Recommended: 100-120 characters per line
  - Maximum: ~140 characters (hard limit is readability)
  - Reason: PowerShell + Windows + UTF-8 combine poorly at high line counts

STRUCTURE.md Regel 8: Naming Conventions
  - Functions: Verb-Noun format (Get-X, Set-X, Test-X, New-X, etc.)
  - Parameters: PascalCase ($ComputerName, $ProfileName)
  - Variables: camelCase ($computerName, $profileName)

STRUCTURE.md Regel 9: Error Handling
  - Use Try-Catch for external resources and known error sources
  - Set ErrorActionPreference = 'Stop' in begin block
  - Use Write-Error for non-terminating errors
  - Log all errors via Write-Log
